# Chapter 7: The Transport Layer

The **Transport Layer** is GAIA's "Network Interface Card." It is responsible for Phase 7 (Dispatch) of the control loop, where abstract requests are turned into protocol-specific packets and sent over the wire.

---

## 7.1 Multi-Protocol Abstraction

GAIA is designed to be transport-agnostic. The Kernel doesn't know (or care) how an agent is connected. It simply calls `invoke(agent, request)`. 

The Transport Layer uses **Adapters** to handle the specific handshake and wire-format for different protocols.

### Supported Protocols:
| Protocol | Wire Format | Best For |
| :--- | :--- | :--- |
| **Native** | JSON-RPC / HTTP | Local or specialized GAIA agents. |
| **A2A** | JSON-RPC 2.0 / HTTPS | Google ecosystem agents. |
| **MCP** | JSON-RPC 2.0 / stdio or SSE | Anthropic ecosystem tools. |
| **IPC** | Shared Memory / Function Call | Ultra-fast local kernel modules. |

---

## 7.2 The Native Protocol

The Native Protocol is the most efficient way to talk to GAIA. It uses standard HTTP `POST` requests for synchronous actions and **WebSocket/SSE** for asynchronous events.

*   **Endpoint**: `agent.endpoint` (provided during handshake).
*   **Auth**: Typically **mTLS** or a **Bearer Token** generated by the Kernel.
*   **Header**: `X-Agent-ID` (used to route the response back to the correct managed state partition).

---

## 7.3 Adapter Architecture

Each protocol adapter is responsible for four tasks:

1.  **Translate**: Convert the GAIA `Request` object into the protocol's format (e.g., A2A `Agent Card` invocation).
2.  **Transmit**: Manage the network connection (Retries, Timeouts, Keep-alive).
3.  **Receive**: Parse the response and map protocol-specific status codes to GAIA's **Error Schema**.
4.  **Normalize**: Convert the protocol-specific output (e.g., MCP content array) back into a standard GAIA JSON `output`.

---

## 7.4 Async Invocation Handling

GAIA supports long-running agent tasks through a two-step async flow:

1.  **Dispatch**: The Kernel sends the request with `mode: "async"`.
2.  **ACK**: The agent returns an immediate `202 Accepted` with a `job_id`.
3.  **Completion**: The agent sends a separate `STEP_COMPLETED` event back to the Kernel's event gateway.

The Transport Layer manages the **correlation** of these events, ensuring that the results of a 30-minute processing job are mapped back to the correct task and step.

---

## 7.5 Isolation & Egress Control

The Transport Layer also acts as a security gate. It enforces **Egress Policies**:
*   **Local Only**: Some agents are forbidden from making outbound network calls.
*   **Whitelisted**: Agents may only talk to specific, pre-approved URLs.
*   **Deny-by-Default**: Unless explicitly granted in the `AgentManifest`, an agent has no network rights beyond communicating with the Kernel.
